# Open WebUI 一键接入工具

> 输入你的 API Key 和中转站地址，一键启动本地 Open WebUI，支持 ChatGPT、Claude、DeepSeek 等所有兼容 OpenAI 格式的模型。

## ✨ 功能特点

- **一键配置**：交互式向导，只需输入 API Key 和中转站地址
- **自动安装**：自动检测 conda 环境并安装 open-webui
- **多模型支持**：支持所有兼容 OpenAI `/v1/chat/completions` 格式的模型
- **本地私有**：所有数据存储在本地，聊天记录不上云
- **无需注册**：可配置跳过登录认证，直接使用
- **知识库支持**：可上传文件让 AI 读取分析

| 要求 | 说明 |

## 🚀 新手使用教程（3 步上手）

### 📥 第一步：获取本项目

从 GitHub 下载项目文件：

1. 打开本仓库页面
2. 点击绿色 **Code** 按钮 → 选择 **Download ZIP**
3. 解压到一个你记得住的文件夹（比如 `D:\openwebui-oneclick\`）

或者用 Git 克隆：
```bash
git clone https://github.com/wulda9716/ship-t/openwebui-oneclick.git
```

### 🔧 第二步：安装环境（只需做一次）

> ⚠️ 这一步需要电脑上已安装 Anaconda 或 Miniconda

1. 打开解压后的文件夹
2. **双击 `install.bat`**
3. 脚本会自动：
   - 检测你的 Anaconda/Miniconda 安装位置
   - 创建名为 `open-webui` 的独立 Python 环境
   - 安装 open-webui 及所有依赖
4. 看到 **"环境安装成功！"** 提示后，按提示选择是否继续配置

#### 各项填写说明：

| 配置项 | 填什么 | 从哪获取 |
|--------|--------|----------|
| **API 中转站地址** | 中转站的 API 地址 | 中转站提供商会告诉你 |
| **API Key** | 你的密钥字符串 | 注册中转站账号后在后台生成 |
| **默认模型名称** | 你想用的模型名字 | 查看中转站支持的模型列表 |
| **监听端口** | 一般保持默认 8080 不动 | 只有被占用时才改 |
| **关闭登录认证** | 选 y（推荐） | 关闭后打开网页就能用，不用注册 |

> 📌 不知道填什么？问你的中转站提供商要一份「API 使用说明」就行。

### ▶️ 第四步：启动！

1. **双击 `start.bat`**
2. 会弹出一个命令行窗口（**不要关掉它！**）
3. 浏览器会自动打开 `http://127.0.0.1:8080`
4. 首次启动需要等待 30-60 秒初始化数据库，看到聊天界面就成功了！

## 🔄 以后怎么用？

以后每次想用的时候：

```
双击 start.bat → 等 10 秒 → 打开浏览器聊天！
```

不需要再运行 install.bat 和 setup.bat。

如果想换模型或换中转站：
```
删除 .env 文件 → 双击 setup.bat → 重新配置
```

## 🔧 支持的模型和中转站

任何兼容 OpenAI `/v1/chat/completions` 格式都可以接入：

| 中转站 | 地址 | 支持模型 |
|--------|------|----------|

| 其他中转站 | 由提供商提供 | 取决于中转站 |

## 📁 项目结构

```
openwebui-oneclick/
├── install.bat     # 第一步：安装环境（首次使用运行一次）
├── setup.bat       # 第二步：配置 API 信息
├── start.bat       # 第三步：一键启动服务
├── .env.example    # 配置模板（参考用）
├── .gitignore      # Git 忽略规则（保护你的 API Key）
└── README.md       # 本文件
```

> ⚠️ `.env` 文件会在你运行 setup.bat 后自动生成，里面包含你的 API Key。**绝对不要把 .env 发给任何人或上传到 GitHub！**

---

## ⚠️ 注意事项

1. **`.env` 包含 API Key，请勿泄露或上传！**（`.gitignore` 已自动忽略）
2. **命令行窗口不能关**——关了服务就停了
3. **首次启动慢是正常的**，在建立本地数据库，等 30-60 秒刷新页面就好
4. **遇到 429 错误** = 请求太频繁，等 1-2 分钟再试
5. **换电脑后需要重新运行 install.bat**

## 🐛 常见问题排查

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| 双击没反应 / 一闪而过 | 缺少 Anaconda | 先安装 Anaconda 再运行 install.bat |
| 提示 "找不到 conda" | Anaconda 路径不在常见位置 | 编辑 install.bat 手动指定路径 |
| 启动后浏览器空白 | 服务还在加载 | 等 30 秒后手动刷新页面（F5） |
| 页面显示但对话报错 | API 信息有误 | 删除 .env，重新运行 setup.bat |
| 端口 8080 已被占用 | 其他程序占用了 | setup.bat 里换个端口（如 8888） |
| 429 错误 | 请求频率超限 | 等一会儿，或联系中转站客服升级套餐 |
| 怎么切换模型？ | - | 在 Open WebUI 网页左上角选择模型 |

## 🎯 高级用法：使用知识库（RAG）

Open WebUI 可以上传文件让 AI 阅读，步骤：

1. 在左侧边栏点击 **📚 Knowledge（知识库）**
2. 点 **Create Knowledge**，取个名字
3. 点 **Upload Files** 上传你要分析的文件
4. 点 **Save & Process** 处理文件
5. 对话时选择这个知识库，AI 就能基于文件内容回答了
